# notification system — sprint 2

## Outcome
Expose notifications to users through a reliable in-app inbox and preference surface.

## Deliverables
- notification list with loading, empty, and populated states
- unread badge in app shell
- read/unread state handling
- preferences read/write UI
- app shell integration
- accessibility baseline

## Exit criteria
- user can see recent notifications
- unread badge reflects backend unread count
- reading notifications clears badge correctly
- preference toggles persist and take effect immediately
- keyboard-only and screen-reader pass succeeds for core flows
